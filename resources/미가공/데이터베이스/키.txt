# 키

1. 후보키(Candidate Key)

Tuple 을 유일하게 식별할 수 있는 능력을 가진 키
 - 유일성 : 무조건 하나 식별됨
 - 최소성 : 더 작은 요소로 쪼갤 수 없음

2. 기본키 (Primary Key)

후보키 중 선택한 키
 - 개체 무결성 : 릴레이션에서 기본키를 구성하는 속성은 NotNull/유일 하다

3. 대체키 (Alternate Key)
후보키 - 기본키

4. 슈퍼키 (Super Key)
유일성을 만족하는 **속성 또는 속성들의 집합** (집합 속에 최소성/기본성을 만족하는 애 있어도 ㄱㅊ)

5. 외래키 (Foreign Key)
한 릴레이션의 속성 집합이 다른 테이블의 기본키가 되는 것
참조 무결성 : 외래키 값은 Null 이거나, 참조 릴레이션의 기본키 값과 같아야 한다.

주문 테이블(릴레이션)
-------
품목 주문자 id
감자 null
딸기 가붕 -> 주문자 id는 회원 테이블의 기본키이며, 비회원 주문시 null도 됨


도메인 무결성 : 특정 속성의 값은 그 도메인에 속한 값이어야 함 (나이 속성에다가 abc 이런거 못 씀)
