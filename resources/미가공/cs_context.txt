# Context

애플리케이션의 전역적인 현재 상태를 알려주는 인터페이스(경계면)이자 안드로이드 시스템에 의해 구현되는 추상 클래스이다.
애플리케이션의 리소스와 클래스 뿐만 아니라 액티비티 실행, broadcasting과 수신 인텐트에 관한 애플리케이션 레벨 동작에 대한 상위 레벨로부터의 액세스를 지원한다.
즉, 애플리케이션들을 관리하는 시스템에게 애플리케이션의 상태를 제공하거나
애플리케이션에게 하위 액티비티와 서비스의 상태를 제공하는데에 모두 쓰이는 것이다. 이때 하위 요소(서비스 등)은 상위 컨텍스트 또한 참조하며 작동한다
다만, 콘텐츠 프로바이더와 브로드캐스트 수신자는 Context를(정확히는 ContextWrapper) 상속받지 않고 
ApplicationContext와 ActivityContext가 주요한데, 각각의 생명주기를 따라 동작한다
ApplicationContext는 주로 애플리케이션에 걸쳐 싱글톤으로 유지되는 객체에 사용할 수 있으며
ActivityContext는 해당 화면에 속하는 UI 동작에 Context가 필요할 때 사용할 수 있다.
적절한 Context를 사용하지 않으면 메모리 누수가 발생할 수 있기 때문에 주의하여야 한다.

tip : Compose는 LocalContext.current 를 사용하여 컨텍스트에 접근할 수 있다.
